## `manage_segments`

**Manage segments**

Create, list, or archive segments — lines of business (or 'Personal'). Each segment is its own P&L and tax filing (Schedule C, Schedule E, or none for personal), but all share one ledger so they combine at year-end. Income/expense accounts belong to a segment; bank/card accounts stay segment-neutral.

**Input schema:**

```json
{
  "type": "object",
  "properties": {
    "action": {
      "type": "string",
      "enum": [
        "list",
        "create",
        "archive"
      ]
    },
    "name": {
      "type": "string",
      "description": "Segment name, e.g. 'Rental', 'Consulting' (required for create)"
    },
    "taxSchedule": {
      "type": "string",
      "enum": [
        "C",
        "E"

      "description": "US tax form: C = Schedule C (sole prop / gig), E = Schedule E (rental). Omit for personal."
    },
    "id": {
      "type": "string",
      "description": "Segment id (required for archive)"
```

## `split_transaction`

**Add a split transaction**

Record one purchase (or deposit) that splits across multiple categories — e.g. a receipt with both business and personal items. Creates a single balanced journal entry: one leg on the payment account (the bank/card — this is what a statement shows and what reconciliation matches) plus one leg per category. Use this instead of add_transaction when a single statement line maps to several categories. Store the full itemization on the receipt; the legs are per-category roll-ups.

**Input schema:**

```json
{
  "type": "object",
  "properties": {
    "date": {
      "type": "string",
      "pattern": "^\\d{4}-\\d{2}-\\d{2}$"
    },
    "description": {
      "type": "string",
      "minLength": 1
    },
    "paymentAccountId": {
      "type": "string",
      "description": "The bank/card account the money moves through — the single leg a statement shows and reconciliation matches"
    },
    "direction": {
      "type": "string",
      "enum": [
        "expense",
        "income"
      ],
      "default": "expense",
      "description": "expense = a purchase split across expense categories (default); income = a deposit split across income categories"
    },
    "legs": {
      "type": "array",
      "items": {
        "type": "object",
        "properties": {
          "accountId": {
            "type": "string",
            "description": "Category account (expense for a purchase, income for a deposit)"
          },
          "amount": {
            "type": "number",
            "exclusiveMinimum": 0,
            "description": "Positive amount in dollars allocated to this category"
          }
        },
        "required": [
          "accountId",
          "amount"
        ],
        "additionalProperties": false
      },
      "minItems": 1,
      "description": "Category legs; their amounts sum to the transaction total"
    },
    "memo": {
      "type": "string"
    },
    "externalId": {
      "type": "string",
      "description": "Bank-provided id used to deduplicate imports"
    },
    "propertyId": {
      "type": "string",
      "description": "Rental property (Schedule E). Use manage_properties → list."

  "required": [
```

## `import_transactions`

**Import transactions from CSV**

Import a bank/card statement (CSV) for one account, as balanced double-entry transactions. Two-step: run with mode='preview' (default) to parse, deduplicate, and flag possible matches — this writes nothing — then review the plan, decide a category account per row, and call again with mode='commit' and `mappings`. The amount sign is handled per profile; posting direction follows the statement (a charge increases a card's balance, a deposit increases a bank balance). Rows are 2-legged (payment + one category); for a line that splits across categories, skip it here and use split_transaction.

**Input schema:**

```json
{
  "type": "object",
  "properties": {
    "mode": {
      "type": "string",
      "enum": [
        "preview",
        "commit"
      ],
      "default": "preview",
      "description": "preview = parse + dedup + flag matches, no writes (default). commit = write the entries named in `mappings`."
    },
    "profile": {
      "type": "string",
      "enum": [
        "signed-amount",
        "debit-credit",
        "rental-export"
      ],
      "description": "CSV shape. Available: 'signed-amount' (One signed amount column where a negative amount is money out (most checking/debit exports). Columns: Date, Description, Amount.) | 'debit-credit' (Separate Debit and Credit columns (debit = money out, credit = money in); tolerates pre-signed debits. Columns: Date, Description, Debit, Credit. Extra columns are ignored.) | 'rental-export' (A pre-categorized property-management export (RFC-4180: quoted fields, embedded newlines). Amount magnitude is taken absolute; direction comes from the Category column. Category, Sub-Category, and Property are surfaced as categorization hints.)"
    },
    "csv": {
      "type": "string",
      "minLength": 1,
      "description": "Raw CSV text of the statement (including the header row)."
    },
    "paymentAccountId": {
      "type": "string",
      "description": "The bank/card account this statement belongs to — the leg every row's payment posts to. Use manage_accounts → list."
    },
    "windowDays": {
      "type": "integer",
      "minimum": 0,
      "maximum": 31,
      "description": "Date window (± days) for flagging possible matches against existing entries. Default 4."
    },
    "mappings": {
      "type": "array",
      "items": {
        "type": "object",
        "properties": {
          "index": {
            "type": "integer",
            "minimum": 0,
            "description": "0-based row index from the preview"
          },
          "targetAccountId": {
            "type": "string",
            "description": "Category account for this row (expense for spend, income for a deposit)"
          },
          "propertyId": {
            "type": "string",
            "description": "Rental property (Schedule E). Use manage_properties → list."
          },
          "skip": {
            "type": "boolean",
            "description": "Explicitly skip this row"
          }
        },
        "required": [
          "index",
          "targetAccountId"
        ],
        "additionalProperties": false
      },
      "description": "Required for mode='commit': one entry per row to import. Rows without a mapping (or with skip=true) are NOT written."
    }
  },
  "required": [
    "profile",
    "csv",
    "paymentAccountId"
  ],
  "additionalProperties": false,
  "$schema": "http://json-schema.org/draft-07/schema#"
}
```

## `manage_rules`

**Manage categorization rules**

Create, list, delete, or test auto-categorization rules. A rule matches a case-insensitive substring of a transaction's description and either routes it to a category account (action=categorize, optionally tagging a rental property) or marks it to skip on import (action=exclude — e.g. transfers between your own accounts). Higher priority wins; ties break on longer pattern. These rules power the suggestions shown by import_transactions preview.

**Input schema:**

```json
{
  "type": "object",
  "properties": {
    "action": {
      "type": "string",
      "enum": [
        "list",
        "create",
        "delete",
        "test"
      ]
    },
    "pattern": {
      "type": "string",
      "minLength": 1,
      "description": "Case-insensitive substring matched against the description (required for create)"
    },
    "ruleAction": {
      "type": "string",
      "enum": [
        "categorize",
        "exclude"
      ],
      "description": "What a matched line does: categorize (needs accountId) or exclude (skip on import). Default categorize."
    },
    "accountId": {
      "type": "string",
      "description": "Target category account — required when ruleAction=categorize; omit for exclude. Use manage_accounts → list."
    },
    "propertyId": {
      "type": "string",
      "description": "Rental property to tag matched entries (Schedule E). categorize only. Use manage_properties → list."
    },
    "priority": {
      "type": "integer",
      "description": "Higher wins on conflicting matches (default 0)."
    },
    "id": {
      "type": "string",
      "description": "Rule id (required for delete)"
    },
    "description": {
      "type": "string",
      "description": "Sample transaction description to test against the rules (required for test)"
    }
  },
  "required": [
    "action"
  ],
  "additionalProperties": false,
  "$schema": "http://json-schema.org/draft-07/schema#"
}
```
