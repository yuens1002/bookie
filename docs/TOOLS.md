## `manage_segments`

**Manage segments**

Create, list, or archive segments — lines of business (or 'Personal'). Each segment is its own P&L and tax filing (Schedule C, Schedule E, or none for personal), but all share one ledger so they combine at year-end. Income/expense accounts belong to a segment; bank/card accounts stay segment-neutral.

**Input schema:**

```json
{
  "type": "object",
  "properties": {
    "action": {
      "type": "string",
      "enum": [
        "list",
        "create",
        "archive"
      ]
    },
    "name": {
      "type": "string",
      "description": "Segment name, e.g. 'Rental', 'Consulting' (required for create)"
    },
    "taxSchedule": {
      "type": "string",
      "enum": [
        "C",
        "E"
      ],
      "description": "US tax form: C = Schedule C (sole prop / gig), E = Schedule E (rental). Omit for personal."
    },
    "id": {
      "type": "string",
      "description": "Segment id (required for archive)"
    },
    "includeArchived": {
      "type": "boolean"
    }
  },
  "required": [
    "action"
  ],
  "additionalProperties": false,
  "$schema": "http://json-schema.org/draft-07/schema#"
}
```

## `add_transaction`

**Add a transaction**

Record a single transaction as a balanced double-entry. Money flows FROM one account TO another: for spending, from your bank/card to an expense account; for income, from an income account to your bank. Amounts are positive, in dollars.

**Input schema:**

```json
{
  "type": "object",
  "properties": {
    "date": {
      "type": "string",
      "pattern": "^\\d{4}-\\d{2}-\\d{2}$"
    },
    "description": {
      "type": "string",
      "minLength": 1
    },
    "amount": {
      "type": "number",
      "exclusiveMinimum": 0,
      "description": "Positive amount in dollars"
    },
    "fromAccountId": {
      "type": "string",
      "description": "Source account (credited / decreased) — e.g. your checking for an expense"
    },
    "toAccountId": {
      "type": "string",
      "description": "Destination account (debited / increased) — e.g. an expense category"
    },
    "memo": {
      "type": "string"
    },
    "externalId": {
      "type": "string",
      "description": "Bank-provided id used to deduplicate imports"
    },
    "propertyId": {
      "type": "string",
      "description": "Rental property this entry belongs to (for Schedule E). Use manage_properties → list."
    }
  },
  "required": [
    "date",
    "description",
    "amount",
    "fromAccountId",
    "toAccountId"
  ],
  "additionalProperties": false,
  "$schema": "http://json-schema.org/draft-07/schema#"
}
```

## `split_transaction`

**Add a split transaction**

Record one purchase (or deposit) that splits across multiple categories — e.g. a receipt with both business and personal items. Creates a single balanced journal entry: one leg on the payment account (the bank/card — this is what a statement shows and what reconciliation matches) plus one leg per category. Use this instead of add_transaction when a single statement line maps to several categories. Store the full itemization on the receipt; the legs are per-category roll-ups.

**Input schema:**

```json
{
  "type": "object",
  "properties": {
    "date": {
      "type": "string",
      "pattern": "^\\d{4}-\\d{2}-\\d{2}$"
    },
    "description": {
      "type": "string",
      "minLength": 1
    },
    "paymentAccountId": {
      "type": "string",
      "description": "The bank/card account the money moves through — the single leg a statement shows and reconciliation matches"
    },
    "direction": {
      "type": "string",
      "enum": [
        "expense",
        "income"
      ],
      "default": "expense",
      "description": "expense = a purchase split across expense categories (default); income = a deposit split across income categories"
    },
    "legs": {
      "type": "array",
      "items": {
        "type": "object",
        "properties": {
          "accountId": {
            "type": "string",
            "description": "Category account (expense for a purchase, income for a deposit)"
          },
          "amount": {
            "type": "number",
            "exclusiveMinimum": 0,
            "description": "Positive amount in dollars allocated to this category"
          }
        },
        "required": [
          "accountId",
          "amount"
        ],
        "additionalProperties": false
      },
      "minItems": 1,
      "description": "Category legs; their amounts sum to the transaction total"
    },
    "memo": {
      "type": "string"
    },
    "externalId": {
      "type": "string",
      "description": "Bank-provided id used to deduplicate imports"
    },
    "propertyId": {
      "type": "string",
      "description": "Rental property (Schedule E). Use manage_properties → list."
    }
  },
  "required": [
    "date",
    "description",
    "paymentAccountId",
    "legs"
  ],
  "additionalProperties": false,
  "$schema": "http://json-schema.org/draft-07/schema#"
}
```

## `query_transactions`

**Input schema:**


## `delete_transaction`

**Delete a transaction**

Permanently delete a journal entry and its postings (used to correct mistakes or remove duplicate imports). Returns what was deleted. This cannot be undone.

**Input schema:**

```json
{
  "type": "object",
  "properties": {
    "entryId": {
      "type": "string",
      "description": "Journal entry id (from add_transaction or query_transactions)"
    }
  },
  "required": [
    "entryId"
  ],
  "additionalProperties": false,
  "$schema": "http://json-schema.org/draft-07/schema#"
}
```
